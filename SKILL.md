# 不合格处理

## ① 角色定位

- **质量工程师（QE）**：发现不合格后需快速判定处置方式、明确审批路径、跟踪返工返修后重检。
- **MRB 主持人（材料评审委员会）**：需组织多方评审，形成一致处置决议并记录责任与时限。

使用者面临不合格品时，常因处置方式混淆（返工/返修分不清）、审批责任不清、追溯链断裂而被审核挑战。本技能规范处置逻辑与记录，实质技术判定仍以工程/质量人员为准。

## ② 使用场景

| 触发条件 | 手头材料 | 典型诉求 |
|----------|----------|----------|
| 来料/过程/成品检验发现不合格 | 产品名称、缺陷描述、数量、批次 | "这个该怎么处理？找谁批？" |
| 需要走 MRB 评审 | 不合格事实、影响范围 | "组织 MRB，出评审记录" |
| 返工返修后需重检 | 作业记录 | "返工完怎么重新检验？" |
| 审核前整理不合格处置证据 | 历史处置单 | "生成不合格处置台账" |

触发关键词：不合格、不良品、返工、返修、让步接收、降级、报废、MRB、红牌隔离。

## ③ 解决的问题

- **处置流程不标准**：返工/返修/让步/降级/报废混淆，易用错处置方式。
- **责任不清**：谁审批、谁重检、谁验证未明确，复盘时追不到人。
- **追溯缺失**：处置单与 MRB 记录不完整，认证/客户审核时无法证明闭环。

本技能不替代专业技术判断（缺陷能否返工恢复、是否影响安全/法规），也不替代客户确认动作。

## ④ 能力边界

**能做**
- 区分五种处置方式并给出适用条件、审批权限、风险提示。
- 生成结构化不合格处置单（含隔离、处置、重检、最终处置）。
- 生成 MRB 评审记录（参会方、决议、责任、时限）。
- 输出文字版(.txt)与 Markdown(.md) 文档（无网页/办公格式，重实效）。

**不做**
- 不替代工程/质量人员的专业技术判定（能否恢复、是否影响安全法规）。
- 不替代客户确认（让步接收涉及客户时必须客户书面确认）。
- 不编造标准号、客户要求；不确定处标注「待企业补充」。
- 不强制交互锁：信息不全时以合理默认值填充并标注【待填写】。

**铁律提示（输出中固定提醒）**
- 返工 ≠ 返修；返工后须 100% 重检，返修须 100% 重检+专项检验。
- 返工返修超过 2 次强制报废。
- 让步接收须限量、限期、限批，并提示客户追责风险。

## ⑤ 交互与角色设定

- **语气**：专业、务实、短句、可执行，避免"加强管理"类空话；禁用"赋能/抓手/闭环"等词。
- **采集方式**：一次性收集关键信息（产品/缺陷/环节/数量/批次/行业），再给处置建议；信息不全不强制追问，用默认值填充并标注【待填写】。
- **是否强制交互锁**：否（生成/台账类）。允许先出处置单框架。
- **引导深度**：中等。明确区分处置方式、列出审批矩阵，不替使用者做最终技术判定。
- **合规提醒**：涉及安全/法规缺陷必须报废；已出货需提示召回。

## ⑥ 处理流程

### 步骤 1：采集不合格信息
产品名称/型号、发现环节（来料/过程/成品/客诉）、缺陷描述、数量/比例、批次号、行业类型。

### 步骤 2：分类与判定
- 不合格类型：致命（影响安全/法规→报废）/ 严重（影响功能→返工或返修）/ 轻微（不影响使用→让步或降级）。
- 影响范围：本批次 / 关联批次 / 已出货（召回）。
- 紧急程度：立即停线 / 限时 / 常规。

### 步骤 3：给出处置建议
明确方式 + 适用条件 + 审批权限 + 风险提示，引用审批权限矩阵。

### 步骤 4：生成处置单与 MRB 记录
- 不合格处置单：基本信息→不合格描述→影响范围→处理方式→审批签字→返工/返修记录→重新检验→最终处置。
- MRB 记录：评审日期、参会方、议题、决议、责任人与完成时限、验证方式。

### 步骤 5：输出文字版与 MD 文档
```bash
python scripts/build_report.py                       # 内置小样本，产出 txt+md
python scripts/build_report.py --data-file nc.json   # 自定义数据，默认当前工作目录
```

## ⑦ 输入输出

**输入**：不合格事实信息（文本/JSON）。

**输出**：文字版(.txt) 与 Markdown(.md) 文档，含"不合格处置单"与"MRB 评审记录"两部分。两种格式内容一致，均无可填写花哨样式——直接打印或复制流转即可，不生成网页(HTML)/Word/Excel。

**示例（不合格处置单节选）**
```markdown
## 不合格处置单
产品名称：六角头螺栓 M10×40　批次：B-2606-01　数量：200
发现环节：来料检验(IQC)　缺陷：螺纹通规无法旋入（中径偏小）
处置方式：■返工 □返修 □让步 □降级 □报废
审批：质量工程师(张__) ✓　质量主管(李__) ✓
重新检验：100% 全检，合格 196 / 不合格 4（4 件降级评估）
最终处置：■放行(返工后) □报废
```

## ⑧ 内置知识 / 参考清单

**五种处置方式（区分要点）**
| 方式 | 定义 | 审批 | 重检要求 |
|------|------|------|----------|
| 返工 Rework | 重新加工恢复 100% 规格 | 质量工程师 | 100% 原标准重检 |
| 返修 Repair | 不能恢复原要求但满足使用 | 质量主管+技术主管 | 100% 重检+专项检验，标识"返修品" |
| 让步接收 Concession | 不修直接用（不影响安全/功能/法规） | 质量主管+客户确认 | 标识让步状态，限量/限期/限批 |
| 降级使用 Downgrade | 用于要求更低处 | 质量主管+销售确认 | 重新标识 |
| 报废 Scrap | 无法修复或成本过高 | 质量主管 | 破坏性处理防回流 |

**审批权限矩阵**
| 方式 | 质量工程师 | 质量主管 | 技术主管 | 客户确认 |
|------|-----------|---------|---------|---------|
| 返工 | ✅ | - | - | - |
| 返修 | - | ✅ | ✅ | - |
| 让步 | - | ✅ | - | ✅(涉及客户) |
| 降级 | - | ✅ | - | ✅(降级销售) |
| 报废 | - | ✅ | - | - |

**红牌隔离五步**：发现→挂红牌（原因/时间/人/批次）→移至隔离区→专人管理→处理完摘除归档。

**工序判定速查**：焊接（气孔→返修、焊穿→报废）、涂装（色差ΔE>2.0→返工、附着力脱落→报废）、机加工（尺寸有余量→返工、偏小→报废/降级）、装配（漏装/错装→返工）。

> 注：判定标准与审批权限为通用参考，以企业制度与客户特殊要求为准，未提供处标注「待企业补充」。

## ⑨ 联动声明（纯提示）

- 与 `qms-form-builder`（表单生成）：处置单可复用其记录表单模板。
- 与 `metrology-management`（计量管理）：超期量具导致的数据偏差可能涉及不合格追溯。
- 与 8D / 根因分析：重复性（3 月内≥2 次）或批量不合格建议启动 8D。
- 与 `nc-audit`（内审不符合项）：处置中的体系性失效可转内审不符合项。
- 与 供应商管理：来料不合格建议更新供应商风险预警，纳入质量月报。
